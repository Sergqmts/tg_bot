Bot API Documentation

🔑 Authentication

Each bot has a unique token. Include it in the URL path:

POST /bot<token>/<method>

The token is generated when you create a bot. You can regenerate it in bot settings.

📨 Messages

sendMessage

Send a text message to a chat or direct message.

POST /bot<token>/sendMessage
{
  "chat_id": 123,        // chat ID, user ID, or @username
  "text": "Hello!"
}
sendPhoto / sendVideo / sendVoice / sendDocument

Send media files (multipart/form-data with file in media field).

POST /bot<token>/sendPhoto
chat_id: 123
media: <file>
caption: "Optional caption"
forwardMessage

Forward a message from one chat to another.

POST /bot<token>/forwardMessage
{
  "chat_id": 123,
  "from_chat_id": 456,
  "message_id": 789
}
deleteMessage

Delete a message sent by the bot.

POST /bot<token>/deleteMessage
{
  "chat_id": 123,
  "message_id": 456
}

🏘️ Community Management

getCommunity
GET /bot<token>/getCommunity?community_id=news
getCommunityMembers
GET /bot<token>/getCommunityMembers?community_id=news
joinCommunity

Bot joins a community as admin.

POST /bot<token>/joinCommunity
{
  "community_id": "news"    // slug or numeric id
}
sendPost

Create a post in a community (bot must be admin).

POST /bot<token>/sendPost
{
  "community_id": "news",   // slug or numeric id
  "body": "Post text here"
}

Supports file uploads via media field (multipart/form-data).

deletePost
POST /bot<token>/deletePost

  "post_id": 123
}

kickMember

  "user_id": 123

promoteToAdmin

🤖 Bot Info & Webhooks

getMe

Get bot profile info.

GET /bot<token>/getMe
setWebhook

Set a URL to receive events (new messages, etc.).

POST /bot<token>/setWebhook
{
  "url": "https://example.com/webhook"
}
deleteWebhook
POST /bot<token>/deleteWebhook

📡 Webhook Events

When a webhook is set, the bot receives POST requests on new messages in Telegram-style format:

{
  "update_id": 1,
  "message": {
    "message_id": 123,
    "from": {"id": 456, "username": "user"},
    "chat": {"id": 789, "type": "group"},
    "text": "Hello bot!"
  }
}

Webhooks fire for group chats (where bot is a member) and DMs (where recipient is the bot). Bot's own messages are excluded.

📝 Quick Start

1. Create a bot at /bots/new — username must end with bot

2. Save the token — it's shown once after creation

3. Test the API:

curl https://socnet.up.railway.app/bot<TOKEN>/getMe

4. Join a community and start posting:

curl -X POST https://socnet.up.railway.app/bot<TOKEN>/joinCommunity \
  -d '{"community_id": "news"}'

curl -X POST https://socnet.up.railway.app/bot<TOKEN>/sendPost \
  -d '{"community_id": "news", "body": "Hello!"}'
