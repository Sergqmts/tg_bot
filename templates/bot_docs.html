Bot API Documentation

Authentication

Each bot has a unique token. Include it in the URL path:

POST /bot<token>/<method>

The token is generated when you create a bot. You can regenerate it in bot settings.

Messages

POST sendMessage

Send a text message to a chat or direct message.

POST /bot<token>/sendMessage
{
  "chat_id": 123,        // chat ID, user ID, or @username
  "text": "Hello!"
}
POST sendPhoto / sendVideo / sendVoice / sendDocument

Send media files (multipart/form-data with file in media field).

POST /bot<token>/sendPhoto
chat_id: 123
media: <file>
caption: "Optional caption"
POST forwardMessage

Forward a message from one chat to another.

POST /bot<token>/forwardMessage
{
  "chat_id": 123,
  "from_chat_id": 456,
  "message_id": 789
}
POST deleteMessage

Delete a message sent by the bot.

POST /bot<token>/deleteMessage
{
  "chat_id": 123,
  "message_id": 456
}
POST receiveMessage internal

Trigger the bot's command handler programmatically — as if a user sent it a DM. For bots with interactive logic (e.g. NewsBot), this processes commands like /start, /help, /bug, /feedback and sends a reply back to the user. Called automatically by the platform when a user messages the bot directly; you can also call it externally. 

POST /bot<token>/receiveMessage
{
  "user_id": 123,      // ID of the user whose message to process
  "text": "/help"      // text to handle (command or free-form)
}

Returns {"ok": true}. The bot's reply is sent as a DM to the user — not included in this response.

Webhook Events

When a webhook is set, the bot receives POST requests on new messages in Telegram-style format:

{
  "update_id": 1,
  "message": {
    "message_id": 123,
    "from": {"id": 456, "username": "user"},
    "chat": {"id": 789, "type": "group"},
    "text": "Hello bot!"
  }
}

Webhooks fire for group chats (where bot is a member) and DMs (where recipient is the bot). Bot's own messages are excluded.

Interactive Commands

The platform automatically routes incoming DMs to your bot's receiveMessage handler. Use setWebhook to receive these events at your server, or implement in-platform command handling by calling receiveMessage directly.

Built-in NewsBot commands

NewsBot — the platform's built-in community bot — handles these commands automatically. All messages sent to it via DM are processed in real time.

Command Description Forwarded to admins?
/start Greeting + feature overview —
/help List of available commands —
/bug <description> Report a bug ✓ all staff
/feedback <text> Leave feedback or a suggestion ✓ all staff
any message Free-form — treated as feedback ✓ all staff

How incoming messages are dispatched

When a user sends a DM to any bot, the platform:

  1. Saves the message to the DB (sender = user, recipient = bot)
  2. If the bot has a webhook set — fires a POST with the message payload
  3. If the bot is NewsBot — also runs the built-in command handler automatically

To add interactive logic to your own bot: set a webhook and handle commands server-side, then reply by calling sendMessage with your bot token.

Example: webhook payload for an incoming DM

{
  "update_id": 42,
  "message": {
    "message_id": 101,
    "from": {"id": 5, "username": "alice"},
    "chat": {"id": 200, "type": "direct"},
    "text": "/help"
  }
}

Respond to this by calling POST /bot<token>/sendMessage with chat_id = the sender's user ID.

Quick Start

1. Create a bot at /bots/new — username must end with bot

2. Save the token — it's shown once after creation

3. Test the API:

curl {{ request.host_url }}bot<TOKEN>/getMe

4. Join a community and start posting:

curl -X POST {{ request.host_url }}bot<TOKEN>/joinCommunity \
  -d '{"community_id": "news"}'

curl -X POST {{ request.host_url }}bot<TOKEN>/sendPost \
  -d '{"community_id": "news", "body": "Hello!"}'

5. Add the bot to a group chat — it will appear in the member list with 🤖
